Telegram-bot как интерфейс к HTTP-API

 

План урока

 
1
Использование HTTP-API в телеграм-ботах
 
2
Бот-переводчик
 
3
Бот-геокодер
 

Аннотация

На этом уроке мы объединим темы разработки telegram-ботов и использования http-api. Телеграм-бот работает как интерфейс между пользователем и http-api для удобного доступа к возможностям последних. Создадим несколько ботов, отвечающих на геопоисковые запросы, отдающих карты и другие результаты работы различных http-api.

 

1. Использование HTTP-API в телеграм-ботах

 

Идея проста: научим телеграм-бота «ходить» в HTTP API, превращая запросы пользователей в http-запросы к API, и транслировать ответы API в удобной и понятной для пользователя форме.

 

2. Бот-переводчик

 

Cоздадим простого бота, который использует API Яндекс.Переводчика для перевода слов с английского на русский язык.

  1. Для начала надо познакомиться с API Переводчика. Документацию вы найдете на этой странице. Обратите внимание на требование сопровождать перевод текстом: Переведено сервисом «Яндекс.Переводчик» http://translate.yandex.ru/.

  2. Получите здесь бесплатный ключ.

  3. Посмотрите описание запроса на перевод тут.

Код функции-переводчика получитcя примерно следующий:

def translater(bot, updater):
    accompanying_text = \
        "Переведено сервисом «Яндекс.Переводчик» http://translate.yandex.ru/."
    translator_uri = \
        "https://translate.yandex.net/api/v1.5/tr.json/translate"
    response = requests.get(
        translator_uri,
        params={
            "key":
            # Ключ, который надо получить по ссылке в тексте.
            "{YOUR_KEY_RECIEVED}",
            # Направление перевода: с русского на английский.
            "lang": "ru-en",
            # То, что нужно перевести.
            "text": updater.message.text  
        })
    updater.message.reply_text(
        "\n\n".join([response.json()["text"][0], accompanying_text]))

 

3. Бот-геокодер

 

Теперь создайте бота, который по запросу пользователя присылает ему карту с запрошенным объектом. Например, пользователь запрашивает: «г. Москва, Тверская улица». В ответ высылается картинка с картой улицы.

Алгоритм работы может быть такой:

  1. Бот «идёт» с адресом в геокодер, получает координаты и размеры окна карты с нужным объектом.

  2. С полученными координатами бот «идёт» в StaticAPI и запрашивает по ним картинку карты.

Необходимо разобраться, как отправлять изображения (фотографии, в нотации телеграм-бота). Об этом можно почитать здесь.

Функция, обрабатывающая сообщения, получится примерно следующая:

def geocoder(bot, updater):
    geocoder_uri = geocoder_request_template = \
        "http://geocode-maps.yandex.ru/1.x/"
    response = requests.get(geocoder_uri, params={
        "format": "json",
        "geocode": updater.message.text
    })
 
    toponym = response.json()["response"]["GeoObjectCollection"][
        "featureMember"][0]["GeoObject"]
    ll, spn = get_ll_spn(toponym)
    # Можно воспользоваться готовой фукнцией,
    # которую предлагалось сделать на уроках, посвящённых HTTP-геокодеру.
 
    static_api_request = \
        "http://static-maps.yandex.ru/1.x/?ll={ll}&spn={spn}&l=map".format(
            **locals)
 
    bot.sendPhoto(
        updater.message.chat.id,  # Идентификатор чата. 
        # Куда посылать картинку.
        # Ссылка на static API, по сути, ссылка на картинку.
        static_api_request
    )
    # Телеграму можно передать прямо её, не скачивая предварительно карту.
